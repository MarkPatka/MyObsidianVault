### 2.1 Принцип работы
 
Алгоритм делит массив на две логические части: **отсортированную** (слева) и **неотсортированную** (справа). Изначально отсортированная часть — это только первый элемент. На каждом шаге берётся первый элемент из неотсортированной части (назовём его `key`) и **вставляется на нужное место** в отсортированной части, сдвигая бо́льшие элементы вправо.
 
Пошагово:
 
1. Считаем `arr[0]` отсортированной частью.
2. Берём `arr[1]` как `key`. Сравниваем его с элементами отсортированной части справа налево.
3. Каждый элемент, который больше `key`, сдвигаем на одну позицию вправо.
4. Когда находим элемент ≤ `key` (или дошли до начала массива) — ставим `key` на освободившееся место.
5. Повторяем для `arr[2]`, `arr[3]`, ..., `arr[n-1]`.
 
Аналогия: так человек сортирует карты в руке — берёт новую карту и вставляет её между нужными.
 
### 2.2 Блок-схема
 
```plantuml
@startuml
start
 
:i = 1;
 
while (i < длина массива?) is (да)
  :key = arr[i]
  j = i - 1;
 
  while (j >= 0 И arr[j] > key?) is (да)
    :arr[j+1] = arr[j]
    j = j - 1;
  endwhile (нет)
 
  :arr[j+1] = key;
  :i = i + 1;
endwhile (нет)
 
stop
@enduml
```
 
### 2.3 Реализация на C#
 
```csharp
public static void InsertionSort(int[] arr)
{
    // Начинаем со второго элемента — первый «уже отсортирован»
    for (int i = 1; i < arr.Length; i++)
    {
        int key = arr[i];   // Элемент, который нужно вставить
        int j = i - 1;      // Начинаем сравнение с конца отсортированной части
 
        // Сдвигаем элементы, которые больше key, на одну позицию вправо
        while (j >= 0 && arr[j] > key)
        {
            arr[j + 1] = arr[j]; // Сдвиг вправо
            j--;                  // Двигаемся влево по отсортированной части
        }
 
        // Вставляем key на найденную позицию
        arr[j + 1] = key;
    }
}
```
 
### 2.4 Анализ сложности
 
| Случай           | Временная сложность | Пояснение                                          |
|------------------|--------------------|----------------------------------------------------|
| **Лучший**       | O(n)               | Массив уже отсортирован — каждый элемент сразу на месте |
| **Средний**      | O(n²)              | Элементы в случайном порядке                        |
| **Худший**       | O(n²)              | Массив отсортирован в обратном порядке              |
 
**Пространственная сложность:** O(1) — in-place, нужна только переменная `key`.
 
**Устойчивость:** Да — равные элементы не переставляются.
 
### 2.5 Область применения
 
- **Малые массивы (до ~50 элементов):** На практике быстрее Quick Sort и Merge Sort из-за минимальных накладных расходов (нет рекурсии, нет дополнительной памяти).
- **Почти отсортированные данные:** Если большинство элементов уже на своих местах, внутренний цикл `while` почти не выполняется — сложность стремится к O(n).
- **Онлайн-сортировка (потоковые данные):** Может сортировать элементы по мере их поступления — не нужен доступ ко всему массиву сразу.
- **Как вспомогательный алгоритм:** Используется внутри Timsort (стандартная сортировка в Python и Java) для сортировки небольших подмассивов.
- **Связанные списки:** Вставка в середину списка — O(1), что делает Insertion Sort естественным выбором для списков.
