### 1.1 Принцип работы
 
Алгоритм многократно проходит по массиву слева направо, сравнивая **каждую пару соседних элементов**. Если левый элемент больше правого — они меняются местами. После первого полного прохода самый большой элемент «всплывает» (как пузырёк) в конец массива. После второго прохода второй по величине элемент занимает предпоследнюю позицию, и так далее.
 
Пошагово:
 
1. Начинаем с первого элемента, сравниваем `arr[0]` и `arr[1]`.
2. Если `arr[0] > arr[1]` — меняем их местами.
3. Переходим к паре `arr[1]` и `arr[2]`, повторяем сравнение и возможный обмен.
4. Дойдя до конца массива, получаем один отсортированный элемент в конце.
5. Повторяем проходы, каждый раз уменьшая рабочую зону на один элемент.
6. Если за весь проход не произошло ни одного обмена — массив отсортирован, выходим досрочно.
 
### 1.2 Блок-схема
 
```plantuml
@startuml
start
 
:swapped = true
n = длина массива;
 
while (swapped?) is (да)
  :swapped = false
  i = 0;
 
  while (i < n - 1?) is (да)
    if (arr[i] > arr[i+1]?) then (да)
      :Обменять arr[i] и arr[i+1]
      swapped = true;
    endif
 
    :i = i + 1;
  endwhile (нет)
 
  :n = n - 1;
endwhile (нет)
 
stop
@enduml
```
 
### 1.3 Реализация на C\#
 
```csharp
public static void BubbleSort(int[] arr)
{
    int n = arr.Length;
    bool swapped;
 
    // Внешний цикл: повторяем проходы, пока есть обмены
    do
    {
        swapped = false;
 
        // Внутренний цикл: проход по неотсортированной части
        for (int i = 0; i < n - 1; i++)
        {
            // Сравниваем соседние элементы
            if (arr[i] > arr[i + 1])
            {
                // Обмен местами через деконструкцию кортежа
                (arr[i], arr[i + 1]) = (arr[i + 1], arr[i]);
                swapped = true; // Фиксируем факт обмена
            }
        }
 
        // После каждого прохода последний элемент на месте —
        // сужаем зону сравнения
        n--;
 
    } while (swapped); // Если обменов не было — массив отсортирован
}
```
 
### 1.4 Анализ сложности
 
| Случай           | Временная сложность | Пояснение                                          |
|------------------|--------------------|----------------------------------------------------|
| **Лучший**       | O(n)               | Массив уже отсортирован — один проход без обменов   |
| **Средний**      | O(n²)              | Элементы в случайном порядке                        |
| **Худший**       | O(n²)              | Массив отсортирован в обратном порядке              |
 
**Пространственная сложность:** O(1) — сортировка выполняется на месте (in-place), нужны только переменные для обмена.
 
**Устойчивость:** Да — равные элементы сохраняют исходный относительный порядок.
 
### 1.5 Область применения
 
- **Учебные цели:** Идеален как первый изучаемый алгоритм сортировки благодаря простоте.
- **Почти отсортированные данные малого объёма:** Благодаря оптимизации с флагом `swapped`, на почти отсортированных массивах работает близко к O(n).
- **Ситуации, где важна устойчивость и простота кода**, а размер данных заведомо мал (десятки элементов).
- **Не рекомендуется** для массивов более ~1 000 элементов — квадратичная сложность делает его крайне медленным.
